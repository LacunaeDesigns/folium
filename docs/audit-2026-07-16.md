# Folium project audit — 2026-07-16

Full-project audit: bugs, code health, UX gaps, and a prioritized roadmap.
Produced by three parallel audit passes (bugs / code health / UX) with all
critical and high findings independently re-verified against the source.

**Baseline:** `tsc -b --noEmit` clean · `npm test` 22 files, 211/211 passing · `main` @ `a8f7463`.

## 1. Bugs (ranked by severity)

### B1 — `applyBackup` wipes all local blobs before validating the incoming ones · CRITICAL · verified
`src/export/json.ts:65-73`. `db.blobs.clear()` commits before `b64ToBuf` (`atob`) ever
runs on the incoming blobs, and `parseBackup` never checks that blob `b64` strings decode.
One corrupted blob entry (e.g. a cloud-sync client presenting a truncated
`folium-workspace.json` — a case `push()`'s own comment in `sync.ts` anticipates) or a
quota error in `bulkPut` throws *after* the clear: every local image/file blob is
irreversibly deleted while the document still references them. Reachable from folder-sync
`reconcile()` (`src/store/sync.ts:99-114`, no try/catch), `linkFolder()`'s load branch,
and the manual Import backup menu.
**Fix:** decode/stage all incoming blobs first, then swap inside one Dexie `rw`
transaction (`blobs` + `templates`); validate blob decodability in `parseBackup`; add
error handling in `reconcile()`. Effort S.

### B2 — Undo after "Empty Trash" resurrects cards whose blobs were already GC'd · CRITICAL · verified
`src/store/persist.ts:169-178` + `src/cards/blobUrl.ts`. `bindBlobGc` fire-and-forgets
blob deletion when cards vanish from `state.cards`; zundo can still restore those cards
(Ctrl+Z after Empty Trash) — metadata returns, bytes are gone, image renders blank with
no error.
**Fix:** include blob ids referenced by zundo's past/future states in
`computeReferencedBlobIds`, or clear temporal history on permanent deletion. Effort S–M.

### B3 — `hydrate()` from tab-sync / boot folder-sync pollutes the undo stack · CRITICAL · verified
`src/store/tabSync.ts:67-76`, `src/store/sync.ts:117-158`, `src/store/store.ts:824-826`.
`hydrate` is a plain tracked `set()`; zundo pushes a pre-sync entry with no
`temporal.clear()` (and the boot-time `reconcile()` path doesn't reload, unlike its
`visibilitychange`/`reconnect` siblings). First Ctrl+Z after a silent sync reverts to the
stale state, which autosave then re-propagates — can clobber the other tab's/machine's
newer data.
**Fix:** `store.temporal.getState().clear()` (or pause/resume) around every `hydrate`.
Effort S.

### B4 — Stale in-app card clipboard hijacks all external pastes · HIGH · verified
`src/canvas/Canvas.tsx:202-241`. Once any card is copied, `hasClipboard()` stays true;
the window paste handler short-circuits before the focused-form-field check. In-app text
copy clears the clip (`useShortcuts.ts:95`), but anything copied *outside the app* never
does — so pasting external text into a table cell/todo/note silently drops it and spawns
a duplicate card; pasting an OS-clipboard screenshot onto the canvas duplicates the old
card instead of creating an image.
**Fix:** stamp the native clipboard with a marker (custom MIME, e.g.
`application/x-folium-clip`) on card copy; on paste, the in-app clip wins only when the
marker is present — external content otherwise takes priority. Effort S–M.

### B5 — LinesLayer drag gestures have no `pointercancel` handling · MEDIUM
`src/canvas/LinesLayer.tsx:195-307`. Endpoint/curve/body drags only commit and clear
state on `pointerup`; a cancelled gesture (stylus palm-rejection, OS gesture) leaves the
line rendered at the dragged position indefinitely. No data corruption (store never
written). Fix alongside B6/B8 via the shared gesture hook (H2). Effort S with H2.

### B6 — Canvas pan/marquee gestures have no `pointercancel` handling · MEDIUM
`src/canvas/Canvas.tsx:284-359`. A cancelled marquee leaves a stuck overlay +
stale gesture ref that survives subsequent pan gestures. Effort S with H2.

### B7 — HTML/Markdown export silently drops horizontal rules; Markdown loses code-block fences · MEDIUM · verified switch gaps
`src/export/html.ts:175-193`, `src/export/markdown.ts:28-63`. NoteCard's unconfigured
StarterKit enables `horizontalRule` (typing `---`) and `codeBlock` (``` fence); neither
export switch handles `horizontalRule`, and Markdown export flattens `codeBlock` to a
plain paragraph. Content silently missing from shared/printed artifacts (JSON backup is
unaffected). Effort S.

### B8 — A cancelled ink stroke is silently discarded by the next stroke · LOW
`src/canvas/InkLayer.tsx:111-152`. No `pointercancel`; the interrupted stroke sits
uncommitted and is overwritten by the next `pointerdown`. Effort S with H2.

## 2. Code health (ranked by value)

### H1 — `onConnectMove` breaks CardShell memoization on every pan/zoom tick · perf, S · verified
`src/canvas/Canvas.tsx:68-73` — `useCallback` depends on `view`, a fresh object per
pan/zoom step, so every visible `CardShell` re-renders each frame — contradicting the
"stable props" comment at line 61. Fix: read `view` via a ref (the wheel handler in the
same file already does this).

### H2 — The threshold-drag/capture/cancel state machine is duplicated 4× · M
`CardShell.tsx` (drag :178, connect :374, resize :409) + `ColumnCard.tsx:19-83`. Commit
`5f55b1c` is direct evidence of the cost: a cancel-recovery fix made in CardShell had to
be rediscovered in the copy. Extract `usePointerDragGesture` (threshold, capture,
buttons-lost recovery, `pointercancel` reset) — then B5/B6/B8 become call-site wiring,
and one hook is testable where four inlined copies weren't.

### H3 — ~10 card types commit content to the store on every keystroke · S
Only NoteCard debounces (`useDebouncedCommit`, 350ms). TableCard, TodoCard, ColumnCard
title, HeadingCard, LinkCard, StickyCard, ShapeCard, SwatchCard, CommentCard, ImageCard
caption all call `updateContent` per keystroke; zundo has no throttle (`store.ts:855-863`),
so typing a sentence floods the 100-entry undo cap (burying meaningful undos) and shallow-
copies the whole `cards` record per character. Fix: reuse the existing hook.

### H4 — Zero tests on the two most interaction-dense files · M
`Canvas.tsx` (917 lines) and `LinesLayer.tsx` (537) have no tests;
`CardShell.test.ts` covers 3 pure helpers, not the gesture machines that produced 3 of
the last 5 commits as bug fixes. Priority: regression tests for the exact cancel/recovery
edge cases already fixed once, plus the B1–B4 fixes above.

### H5 — Unmemoized whole-document selectors · M · not urgent
`store/selectors.ts` scans and sorts every card in the document on each render of the
open board, for edits anywhere. Fine at current scale; memoize per-board when documents
grow past a few hundred cards.

### H6 — Minor: type-safety spot risks and dead code · S
`persist.ts:155` and `ChartCard.tsx:32` `as unknown as` casts; 27 convention-guarded `!`
assertions (fragile ones around `viewportEl`); `worldToScreen` (`coords.ts:22`) has zero
call sites — delete or keep deliberately for a minimap.

### H6b — Note cards commit on mount · S · found during post-fix verification
Opening a board that contains note cards pushes undo entries (and dirties
autosave) before the user touches anything: TipTap's mount-time normalization
of the stored doc fires `onUpdate`, and NoteCard commits it. Observed live:
a fresh boot of the seed board starts with 2 undo entries. Fix direction:
skip the commit when the normalized doc deep-equals the stored one, or ignore
the first `onUpdate` after mount.

### H7 — Deliberate-upgrade backlog · directional
React 18 / Zustand 4 / Vite 5 / TipTap 2 / Vitest 2 are each ≥1 major behind (not
verified against the live registry). Plan one deliberate upgrade pass rather than drift.

## 3. UX gaps and friction (verified against code)

**Undiscoverable existing features:** align/distribute/z-order exist
(`Canvas.tsx` ~448-515) but are right-click-only — no toolbar presence, no shortcuts, and
absent from the Help panel's own shortcut table. Zoom-to-selection exists (Shift+2) but
isn't on the zoom pill.

**Table-stakes gaps:** no card locking · no rotation · no explicit group command (frames
are geometry-only) · no durable version history (undo dies on reload) · fixed 6–9 color
palettes for Sticky/Shape/Note while Swatch already has free hex · grid is decorative
only (no snap-to-grid) · no PWA manifest/service worker (local Node server required) ·
~1 `aria-label` in the whole tree, no `role=` usage · no touch/pinch support · no
cell-to-cell keyboard nav or TSV paste in Table/Chart · Table/Chart rows/columns
append/remove-at-end only · Ink cards are frozen after creation · search caps at 30 hits
with no filters.

## 4. Roadmap

### Wave A — data safety (first; all small)
1. B1 transactional backup import (the one genuinely scary bug).
2. B3 temporal-clear on every hydrate path.
3. B2 undo-aware blob GC.
4. B4 clipboard freshness marker.
5. From existing backlog: "force push mine" sync-conflict action.

### Wave B — interaction correctness + perf
6. H2 shared `usePointerDragGesture` hook; wire B5/B6/B8 cancel handling through it.
7. H1 `onConnectMove` ref fix.
8. H3 debounced commits across the ~10 card types.
9. B7 export node coverage (`horizontalRule`, Markdown `codeBlock`).
10. H4 regression tests over all of the above.

### Wave C — discoverability + table-stakes UX (mostly S)
11. Align/distribute/z-order → toolbar buttons + shortcuts + Help panel entries.
12. Card locking (`locked` flag, gate move/resize/trash, context-menu toggle + badge).
13. Free hex color for Sticky/Shape/Note (reuse SwatchCard's picker pattern).
14. Table/Chart: Tab/Enter cell nav, TSV bulk paste, insert/reorder rows and columns.
15. Zoom-to-selection on the zoom pill; search type/board filters + "next 30".
16. Opt-in snap-to-grid tied to the existing grid backgrounds.
17. Backlog: dark-mode values for `--card-*` tokens; ARIA pass (help tabs first).

### Wave D — flagship features
18. **Local version snapshots** — dated JSON snapshots (reusing `export/json.ts`) in a
    Dexie table or the synced folder, surfaced as "Version history." The highest-trust,
    most on-brand feature: fixes "undo dies on reload" and softens every data-risk above.
19. All-boards dashboard (grid of `BoardCard` tiles + search-index filtering).
20. Global quick-capture hotkey → Unsorted/Inbox.
21. Ink re-edit (append strokes / erase on existing ink cards).
22. PWA installability (manifest + service worker) — also improves the distribution
    story vs. `start-folium.bat`, and dovetails with the desktop-packaging REVISIT.

### Wave E — longer horizon
23. Sync merge logic for the known last-write-wins conflict friction (or a documented
    one-machine-at-a-time discipline + better conflict UI).
24. Touch/tablet input (pinch zoom, larger hit targets).
25. Dependency major-version upgrade pass.
26. Memoized selectors when board sizes warrant (H5).
